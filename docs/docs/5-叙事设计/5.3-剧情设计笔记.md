# 角色剧情优化计划

> 本文档记录以"角色塑造"为核心的剧情系统优化方向和待办事项。

---

## 优化目标

将剧情从"独立的文本内容"转变为"与玩法深度绑定的角色体验"，让玩家在游戏过程中感受角色、理解角色、喜爱角色。

---

## 已确定的设计方向

### 1. 玩家定位：沉默指挥官

- 玩家有存在感但不说话
- 舰长们围绕玩家展开故事
- 类似《明日方舟》的博士定位

### 2. 剧情核心：角色关系网

- 舰长之间的关系是剧情主线驱动力
- 不是48条独立的玩家-舰长双人线
- 而是角色间交织的群像故事

### 3. 角色分层：核心圈优先

- 先精简角色数量，重点做核心圈
- 核心圈角色有完整故事弧和深度塑造
- 其他角色后续扩展

---

## 核心圈角色选择

### 选择标准

- 数量：7-8人
- 优先级：战舰玩法多样性
- 覆盖流派：平衡、暴击、极端机制、召唤、规避、元素、控制、单武器

### 命名规范

**全角色统一采用"全名+显示名"体系**：

- **显示名**：游戏界面显示，2-3字，简短好记
- **全名**：详情页/档案显示，有文化内涵和故事感
- **设计原则**：
  - 名字暗示角色特点，但不直白
  - 避免"形容词+名字"的中二格式
  - 中式/西式/混合均可，与角色背景匹配

### 角色关系网设计

**已确定的关系**：
- 朱璃 ↔ 玄霜：双胞胎姐妹（火与冰的对立）
- 夜凛 ↔ 莉莉丝：宿敌（待设计具体冲突）

**待设计的关系**：
- 星遥与其他人的关系（作为新人/视角人物）
- 零的立场（与机械叛军主线的关联）
- 希尔薇、莱拉的关系网位置

## 待办事项

### 一、战斗反馈中的角色存在感 [ ]

**目标**：让玩家在战斗中感受角色性格，而非仅在剧情界面

**待讨论内容**：
- [ ] 语音分层设计：好感度不同阶段的语音风格变化
- [ ] 专属台词设计：根据战舰负面属性/特殊机制设计角色专属语音
- [ ] 条件语音触发：根据Build风格、战斗状态触发特定语音
- [ ] 战斗中的性格表达方式（语音频率、语气、内容）

**当前状态**：待讨论

---

### 二、剧情与战舰机制绑定 [ ]

**目标**：形成"人设即玩法"，角色背景解释战舰设计

**待讨论内容**：
- [ ] 核心角色的机制-剧情对照表设计
- [ ] 第1-2章剧情定位：揭示"为何她会成为这样"
- [ ] 战舰负面属性的叙事化解释
- [ ] 玩法体验与角色理解的先后顺序

**当前状态**：待讨论

---

### 三、碎片化叙事系统 [ ]

**目标**：用碎片化叙事替代线性剧情，符合Roguelike节奏

**待讨论内容**：
- [ ] 碎片解锁条件设计（成就型、累计型、挑战型）
- [ ] 碎片内容形式（文本、语音、CG、回忆片段）
- [ ] 碎片拼图机制：收集碎片还原完整故事
- [ ] 与现有5章剧情结构的关系（替代/并存/融合）

**当前状态**：待讨论

---

### 四、角色关系网剧情 [ ]

**目标**：通过角色关系增加收集深度和剧情层次

**待讨论内容**：
- [ ] 现有关系设定扩展（姐妹、师徒、宿敌、搭档等）
- [ ] 跨角色剧情解锁条件（同时拥有、同时使用、好感度组合）
- [ ] 关系剧情的内容形式和篇幅
- [ ] 关系网对主线剧情的影响

**当前状态**：待讨论

---

### 五、角色数量与深度平衡 [ ]

**目标**：在48人规模下实现有效的角色塑造

**待讨论内容**：
- [ ] 核心角色 vs 简化角色的分层策略
- [ ] 垂直切片5艘战舰的角色塑造深度验证
- [ ] 不同层级角色的内容配置（剧情章节、语音数量、CG数量）
- [ ] 商业模式考量（抽卡/解锁/DLC）

**当前状态**：待讨论

## 优先级排序

| 优先级 | 方向 | 理由 |
|--------|------|------|
| P1 | 剧情与战舰机制绑定 | 低成本高回报，奠定角色塑造基础 |
| P2 | 碎片化叙事系统 | 符合Roguelike节奏，核心体验创新 |
| P3 | 战斗反馈中的角色存在感 | 提升沉浸感，需要语音资源 |
| P4 | 角色关系网剧情 | 增加收集深度，依赖角色基础完成 |
| P5 | 角色数量与深度平衡 | 影响整体规划，需要商业决策 |
