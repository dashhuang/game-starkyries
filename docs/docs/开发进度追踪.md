## 🎯 伤害类型系统（三类标签）

> **设计理念**：武器可同时拥有多个伤害类型标签，加成乘算叠加

### 伤害类型定义（三大类 + 工程）

> **设计理念**：参考 Brotato 的 Melee/Ranged/Elemental 分类，结合太空题材重新命名

| 类型 | 判定方式 | 说明 | Brotato对应 |
|------|----------|------|-------------|
| **近程** | 武器类型 = FORCE_FIELD | 力场系列，贴身战斗 | Melee |
| **弹炮** | 武器类型 = MACHINEGUN 或 MISSILE | 机炮+导弹，物理弹道 | Ranged |
| **能量** | 武器类型 = ARC 或 LASER | 电弧+激光，能量效果 | Elemental |
| **工程** | 武器类型 = CARRIER | 舰载机，召唤物 | Turret（无伤害加成） |

### 武器分类示例

| 武器 | 类型 | 伤害加成 |
|------|------|----------|
| 速射力场 | FORCE_FIELD | ×(1+近程) |
| 协同力场 | FORCE_FIELD | ×(1+近程) |
| 电弧冲击 | ARC | ×(1+能量) |
| 粒子机炮 | MACHINEGUN | ×(1+弹炮) |
| 追踪导弹 | MISSILE | ×(1+弹炮) |
| 激光狙击 | LASER | ×(1+能量) |
| 战斗无人机 | CARRIER | 无伤害类型加成（仅受工程%） |

### 对应模块

| 模块 | 效果 | 适用武器 | 属性名 |
|------|------|----------|--------|
| 近战强化 | +20%伤害 | 力场系列 | `meleeDamageBonus` |
| 弹道计算机 | +20%伤害 | 机炮、导弹、鱼雷 | `ballisticDamageBonus` |
| 能量增幅 | +20%伤害 | 电弧、激光 | `energyDamageBonus` |
| 工程专家 | +15%工程 | 舰载机 | `engineering`（不是伤害加成） |

### 设计优势

1. **清晰的分类**：基于武器类型（type字段）判定，无歧义
2. **简化叠加**：每种武器只受一种伤害类型加成，计算简单
3. **Build多样性**：
   - 近战流：力场武器，堆近战强化
   - 弹炮流：机炮+导弹，堆弹道计算机
   - 能量流：电弧+激光，堆能量增幅
   - 工程流：舰载机，堆工程专家（不同的加成机制）

### 2. 武器（10种）| 🔑 按效果类型划分

**VS核心验证目标**：每种武器效果类型都能正确实现 + 体现游戏战斗风格

> **设计理念（对标Brotato）**：
> - 入门武器(15-25晶体)占50%，让玩家低成本体验各种风格
> - 必须包含"叠加流"代表武器（协同力场），这是Brotato最具特色的设计
> - 各武器类别（近程/弹炮/能量/舰载机）都有入门选项

| **叠加** ⭐ | 协同力场 | 4 | 0.58s | 1.5 m | 15 | ⭐每把+12%火力，展示Brotato式叠加流 |

**🔑 武器效果验证清单**：

```
□ 直射：子弹从战舰发射→直线飞行→命中敌舰→造成伤害（速射力场）
□ 叠加：每装备一把协同力场→火力+12%→叠6把=+72%火力（⭐Brotato特色）
□ 弹道：导弹发射→追踪最近敌舰→命中（轻型导弹）
□ 激光：即时命中→无飞行时间→直接造成伤害（训练激光）
□ 击退：命中→敌舰被推开一段距离（冲击脉冲）
□ 召唤：炮塔固定位置→自动攻击射程内敌舰（迷你炮塔）
□ 高射速：0.25秒射一发→屏幕上大量子弹→性能稳定（风暴力场）
□ DOT：命中→敌舰持续掉血3秒→死亡或结束（等离子喷射器）
□ AOE：导弹命中→爆炸范围内所有敌舰受伤（集束导弹）
□ 高伤害：命中→大数字反馈→敌舰明显受创（重型鱼雷）
```

## 📐 空间单位系统

> **设计目标**：分辨率无关，易于理解，便于设计

### 竞技场设计（对标Brotato）⭐已更新

> **Brotato核心设计**：竞技场 > 屏幕，玩家有走位空间但无法无限逃跑
> **星河战姬特色**：支持横屏/竖屏，动态可视区域适配不同屏幕

#### 尺寸定义

| 概念 | 尺寸 | 说明 |
|------|------|------|
| **竞技场（战斗区域）** | **50×30 m** | 横向更宽，支持横竖屏 |
| **可视区域（动态）** | 短边22 m，长边最大40 m | 根据屏幕比例自动计算 |

#### 动态可视区域

可视区域根据屏幕宽高比**动态计算**，保证短边至少看到 22 m：

| 屏幕类型 | 比例 | 可视区域 | 相机移动空间 |
|----------|------|----------|-------------|
| 手机横屏 | 16:9 | ~40×22 m | 左右10 m，上下8 m |
| 手机竖屏 | 9:16 | ~22×40 m | 左右28 m，上下0 m |
| PC超宽屏 | 21:9 | ~40×22 m | 左右10 m，上下8 m |
| 平板 | 4:3 | ~30×22 m | 左右20 m，上下8 m |

> **设计说明**：PC大屏幕可以看到更多内容，这不是竞技优势，而是更好的视觉体验

```
竞技场布局示意图（横屏）：

┌─────────────────────────────────────────────────────────────────────┐
│                          敌舰生成区域                                 │
│    ┌───────────────────────────────────────────────────┐            │
│    │                                                   │            │
│ 敌 │              可视屏幕区域（动态）                   │ 敌         │
│ 舰 │               (~40×22 m)                         │ 舰         │
│ 生 │                 ◄══►                              │ 生         │
│ 成 │                玩家战舰                            │ 成         │
│    │                                                   │            │
│    └───────────────────────────────────────────────────┘            │
│                          敌舰生成区域                                 │
└─────────────────────────────────────────────────────────────────────┘
                      竞技场总尺寸：50×30 m
```

#### 摄像机行为

| 行为 | 说明 |
|------|------|
| **跟随模式** | 摄像机平滑跟随玩家移动 |
| **边界限制** | 可视区域不会超出竞技场边界 |
| **动态适配** | 屏幕旋转/缩放时自动重新计算 |
| **平滑参数** | FollowSmoothing = 5.0 |

#### 敌舰生成规则

| 规则 | 说明 |
|------|------|
| **生成位置** | 竞技场边缘（屏幕外） |
| **生成方向** | 360°全方位包围玩家 |
| **最小距离** | 距离玩家至少8 m（避免突然出现在身边） |

#### 与Brotato对比

| 维度 | Brotato | 星河战姬 | 说明 |
|------|---------|----------|------|
| 竞技场尺寸 | ~1200×800像素 | **50×30 m** | 横向更宽 |
| 可视区域 | 固定960×540像素 | **动态适配** | 支持多种屏幕 |
| 竞技场面积比 | ~1.85倍屏幕 | **~2.7倍**（横屏） | 更大走位空间 |

| 横竖屏支持 | 仅横屏 | **横竖屏均支持** | 移动端友好 |

### 为什么不用像素？

Brotato使用像素单位是因为其固定分辨率（960×540）。但我们需要支持多种分辨率，像素单位会导致：
- 高分辨率下距离看起来更短
- 策划数值难以直观理解
- 玩家无法理解"500像素射程"的含义

### 🔑 虚拟米制（m = 米）

战舰是**横向长方形**（如游戏截图所示），为避免歧义，我们定义：

```
1 米 = 玩家战舰的短边（高度）

战舰示意图（俯视/横版）：
        ← 长边（约3 m）→
       ┌─────────────────┐
1 m →  │  ◄══ 战舰 ══►   │
       └─────────────────┘
```

| 概念 | 尺寸 | 参考截图 |
|------|------|----------|
| **战舰高度（短边）** | **1 m** | 基准单位 |
| **战舰长度（长边）** | **~3 m** | 战舰头尾长度 |
| **能量晶体** | **~0.5 m** | 截图中绿色发光方块 |
| **小型敌舰** | **~1 m** | 截图左上黑色球体 |
| **可视区域短边** | **22 m** | 最小保证可见 |
| **可视区域长边** | **最大40 m** | 根据屏幕比例动态计算 |

### 距离分类

| 距离类型 | 范围 | 直观描述 | 对照截图 |
|----------|------|----------|----------|
| **贴脸** | 0-1 m | 几乎接触 | 敌舰触碰战舰 |
| **近距离** | 1-3 m | 一个战舰长 | 约1个战舰长度 |
| **中距离** | 3-8 m | 几个身位 | 战舰到最近晶体 |
| **远距离** | 8-20 m | 四分之一屏 | 战舰到左上敌舰 |
| **超远** | 20+ m | 半屏以上 | 接近屏幕边缘 |

### 单位换算公式（程序实现）

```
实际渲染距离 = 米数 × 战舰短边渲染尺寸

示例（1080p，战舰高度=48像素）：
- 3 m射程 = 3 × 48 = 144像素

示例（4K，战舰高度=96像素）：
- 3 m射程 = 3 × 96 = 288像素

结果：无论分辨率，射程始终是"3个战舰高度"
```
