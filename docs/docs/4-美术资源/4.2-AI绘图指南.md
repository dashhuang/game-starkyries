# AI 绘图指南

> 本文档记录使用 AI 生成游戏图片资源的规范和最佳实践。

---

## 〇、资源规格总览

### 图片规格速查表

| 资源类型 | 尺寸 | 比例 | 格式 | 背景 | 存放路径 |
|---------|------|------|------|------|---------|
| **武器图标** | 512×512 | 1:1 | JPG Q90 | 黑色/深色 | `assets/images/weapons/{WeaponId}.jpg` |
| **模块图标** | 512×512 | 1:1 | JPG Q90 | 黑色/深色 | `assets/images/modules/{module_id}.jpg` |
| **角色头像** | 512×512 | 1:1 | JPG Q90 | 含背景 | `assets/image/{角色名}/头像.jpg` |
| **角色立绘** | 2048×2048 | 1:1 | JPG Q90 | 含背景 | `assets/image/{角色名}/{表情}.jpg` |
| **场景背景** | 2048×2048 | 1:1 | JPG Q90 | - | `assets/image/scene/` |

### 关键规则

1. **角色立绘是 1:1 正方形**（2048×2048），角色与背景在同一张图上
2. **武器/模块图标用黑色背景**，不要白色背景
3. **命名规范**：武器用 PascalCase（如 `LaserSniper.jpg`），模块用 snake_case（如 `heavy_armor.jpg`）

### 格式转换命令

```bash
# 武器/模块图标
convert input.png -resize 512x512! -background black -flatten -quality 90 output.jpg

# 角色头像
convert input.png -resize 512x512! -quality 90 头像.jpg

# 角色立绘
convert input.png -resize 2048x2048! -quality 90 表情.jpg

# 场景背景（与立绘相同）
convert input.png -resize 2048x2048! -quality 90 scene.jpg
```

## 一、工具说明

使用系统 MCP 工具：`mcp__sce-urhox__generate_image`

### 参数说明

| 参数 | 类型 | 必填 | 限制 | 说明 |
|------|------|------|------|------|
| `prompt` | string | ✅ | 无硬性限制 | 中文描述，可以很长 |
| `name` | string | ✅ | ≤5字符 | 文件名（不含扩展名） |
| `aspect_ratio` | string | ❌ | 见下表 | 宽高比，默认1:1 |
| `transparent` | boolean | ❌ | - | 透明背景 |

> 💡 **提示**：这是多模态绘图模型，prompt 越详细效果越好，建议包含游戏背景、美术风格、具体内容、视觉特征等

### 宽高比选项

| 比例 | 用途 |
|------|------|
| `1:1` | 图标、头像、道具、**角色立绘**、**场景背景**（默认） |
| `16:9` | 宣传图、视频封面（非游戏内资源） |
| `9:16` | 竖屏、短视频封面 |

> ⚠️ **角色立绘必须是 1:1**，角色与背景在同一张图上

### 关于透明背景

- 工具支持透明背景（`transparent: true`）
- **但抠图能力较弱**，边缘可能不够干净
- **建议**：非必要不使用透明背景，改用纯色背景（如黑色太空背景）

## 二、Prompt 设计原则

### 核心原则：信息越丰富越好

这是多模态 AI 绘图模型，**prompt 可以很长**，提供的上下文越丰富，生成效果越好。

### Prompt 应包含的信息

1. **游戏背景**：这是什么游戏，什么题材
2. **美术风格**：参考什么风格，色调是什么
3. **具体内容**：要画什么东西
4. **视觉特征**：形状、颜色、材质、光效
5. **负面提示**：不要什么（如：不要图标边框，不要文字）

### Prompt 结构模板

```
[游戏背景] + [美术风格] + [具体内容和描述] + [视觉特征和配色] + [负面提示]
```

**示例格式**：
```
游戏《星河战姬》是二次元太空战舰3D弹幕游戏，美术风格参考罪恶装备、明日方舟，简洁粗线条日式二次元风格。现在绘制粒子机炮，这是太空战舰的多管旋转速射武器，六管炮管呈环形排列，金属质感枪钢色炮身，发射金色能量粒子弹幕。深蓝星空背景，不要图标边框，不要文字
```

### 关键技巧

#### ❌ 不要说"图标"

```
❌ 错误：生成一个速射力场的武器图标
✅ 正确：生成速射力场这种武器的图片，用于游戏中的武器选择界面
```

> AI 听到"图标"可能会画出图标边框，我们需要的是内容本身

#### ❌ 不要过于简短

```
❌ 错误：力场武器
✅ 正确：[完整的游戏背景 + 武器描述 + 视觉特征 + 用途]
```

## 五、角色图片生成

> 角色设定请参阅：[5.2-角色索引.md](../5-叙事设计/5.2-角色索引.md)
> 详细角色文档：[characters/](../5-叙事设计/characters/)

### ⚠️ 角色图片规格（重要）

| 类型 | 尺寸 | 比例 | 格式 | 说明 |
|------|------|------|------|------|
| **设定原画** | 2048×2048 | 1:1 | PNG/JPG | 角色形象基准，不直接用于游戏 |
| **角色立绘** | 2048×2048 | **1:1** | JPG | 角色+背景在同一张图，用于对话系统 |
| **角色头像** | 512×512 | 1:1 | JPG | 裁剪自立绘或单独生成，用于选择界面 |

> 💡 **关键**：角色立绘必须是 **1:1 正方形**，角色和背景画在一起，不要分开

### 🎨 设定原画规则（重要）

**设定原画**是角色形象的基准参考图，记录了角色的标准外貌、服装、配饰等视觉特征。

#### 用途
- 不直接在游戏中使用
- 作为生成该角色其他图片的**参考图**
- 确保同一角色的所有图片风格一致

#### 存放位置
```
assets/image/{角色名}/设定原画.png
```

#### ⚠️ 生成角色图片时必须参考设定原画

当需要为某个角色生成新图片（立绘、头像、表情变体等）时：

1. **先检查是否有设定原画**：`assets/image/{角色名}/设定原画.png`
2. **如有设定原画**：将其作为参考图传给绘图工具，要求在此基础上创作
3. **如无设定原画**：根据角色文档描述生成

#### ⚠️ 有参考图时 Prompt 要简洁

**当使用设定原画作为参考图时，Prompt 不要包含角色外貌细节**，否则会与参考图冲突，影响一致性。

| 情况 | Prompt 内容 |
|------|------------|
| **有参考图** | 游戏背景 + 表情/情绪 + 场景/背景 + 姿态（简洁） |
| **无参考图** | 游戏背景 + 美术风格 + 外貌细节 + 服装 + 表情 + 场景（详细） |

**有参考图时的 Prompt 示例**：
```
修改为战斗失败，低头沮丧
修改为战斗失败，单膝跪地
修改为战斗失败，握拳不甘心
```

> 💡 **技巧**：用不同动作区分多张图，让模型自由发挥

**❌ 错误（有参考图时）**：
```
...零是AI少女，银白色双马尾发型发尾渐变为青蓝色，蓝色眼睛，穿白色紧身战斗服有青色电路纹路装饰...
```
> 这些外貌细节已经在参考图中，重复描述会导致冲突

### 角色风格要点

- **定位**：英气飒爽的女性舰长
- **服装**：军装风格，科幻元素
- **气质**：冷峻专业，不是萌系

### 角色 Prompt 模板

```
游戏《星河战姬》是二次元太空战舰游戏，美术风格参考罪恶装备、明日方舟，简洁粗线条日式二次元风格。
绘制角色{名称}，{身份}，{外貌描述}，{服装描述}，{表情}。
背景是{场景描述}，不要文字
```

### 角色 Prompt 应包含

1. 游戏背景和世界观
2. 角色身份和性格
3. 外貌特征（发型、发色、眼睛等）
4. 服装描述
5. 表情和姿态
6. **背景环境**（必须，角色和背景在同一张图）
7. 用途（立绘/头像/对话图）

### 表情变体命名

| 文件名 | 用途 |
|--------|------|
| `头像.jpg` | 512×512 头像，用于战舰选择界面 |
| `坚定.jpg` | 默认表情立绘 |
| `微笑.jpg` | 微笑表情 |
| `紧张.jpg` | 紧张表情 |
| `加油.jpg` | 鼓励表情 |

## 六、场景图片生成

### ⚠️ 场景图片规格（重要）

| 类型 | 尺寸 | 比例 | 说明 |
|------|------|------|------|
| **场景背景** | 2048×2048 | **1:1** | 与角色立绘格式相同 |

> 💡 **关键**：场景背景也是 **1:1 正方形**（2048×2048），与角色立绘保持一致

### 常见场景类型

| 场景 | 描述 |
|------|------|
| 战斗背景 | 太空战场、星际舰队交战 |
| 舰桥内部 | 指挥室、控制台 |
| 母港 | 停泊的舰队、空间站 |
| 补给站 | 商店界面背景 |

### 场景 Prompt 应包含

1. 游戏背景
2. 场景类型和氛围
3. 主要元素
4. 光照和色调
5. 用途

## 七、文件命名规范

由于 `name` 参数限制 ≤5 字符，建议使用简短中文：

| 类型 | 命名示例 |
|------|----------|
| 武器 | 速射、穿透、风暴、电弧 |
| 角色 | 星遥、夜凛、零 |
| 场景 | 战场、母港、补给 |
| 道具 | 晶体、护盾、修复 |
