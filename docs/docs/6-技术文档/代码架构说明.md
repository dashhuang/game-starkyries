# 星河战姬 - 代码架构说明

> **版本**：v3.1（敌人配置系统）
> **更新日期**：2026-01-07

---

## 概述

星河战姬采用模块化 Lua 架构，基于 UrhoX 引擎开发。本文档描述包含效果组件系统的代码结构。

### 架构特点

- **单一职责**：每个模块专注一个领域
- **低耦合**：模块间通过回调/事件通信
- **配置集中**：所有数值配置统一在 `config/` 目录
- **延迟加载**：避免循环依赖
- **数据驱动效果**：触发型效果通过配置定义，无需硬编码

## 核心模块说明

### main.lua（入口协调者）

**职责**：初始化、模块加载、回调设置、主循环协调

```lua
-- 主要函数
CreateScene()           -- 创建场景和相机
InitGame()              -- 初始化游戏流程
StartGameWithShip()     -- 开始游戏
SetupCallbacks()        -- 设置模块间回调
HandleUpdate()          -- 主更新循环
HandlePreRenderUI()     -- UI渲染
```

### core/Game.lua（状态中心）

**职责**：管理玩家数据、游戏状态、资源操作

```lua
-- 状态枚举
Game.States = { PLAYING, PAUSED, SHOP, BRIDGE_UPGRADE, GAME_OVER, VICTORY }

-- 玩家数据
Game.player = { shield, crystals, xp, weapons, modules, ... }

-- 关键函数
Game.TakeDamage(damage)     -- 受伤处理（触发规避、装甲、无敌）
Game.DirectDamage(amount)   -- 直接伤害（忽略装甲和无敌，用于自伤效果）
Game.Heal(amount)           -- 治疗
Game.AddCrystals(amount)    -- 获得晶体
Game.AddXp(amount)          -- 获得经验
Game.CheckAndAccumulateUpgrades()  -- 检查升级
```

### core/EffectSystem.lua（效果执行引擎）⭐ 新增

**职责**：监听事件、执行触发型效果

```lua
-- 初始化
EffectSystem.Init()              -- 订阅所有效果的触发事件

-- 核心函数
EffectSystem.CheckTrigger(event, data)  -- 检查并执行效果
EffectSystem.GetStackCount(effectId)    -- 获取效果叠加层数
EffectSystem.GetActiveEffects()         -- 获取激活效果列表

-- 特殊效果触发
EffectSystem.TriggerChainDamage(x, y, damage, radius)  -- 连锁伤害
EffectSystem.TriggerAOEDamage(x, y, damage, radius)    -- AOE 伤害

-- 调试
EffectSystem.SetDebug(enabled)   -- 启用调试输出
EffectSystem.Debug()             -- 打印系统状态
```

### core/GameLoop.lua（游戏循环）

**职责**：武器更新、碰撞检测、实体更新调度

```lua
-- 关键函数
GameLoop.UpdateWeapons(dt, scene)    -- 武器开火逻辑
GameLoop.FireWeapon(...)             -- 武器射击
GameLoop.CheckCollisions(scene)      -- 碰撞检测
GameLoop.UpdatePlayer(dt, mx, my)    -- 玩家更新
GameLoop.UpdateEntities(dt, px, py)  -- 实体更新
GameLoop.UpdateEffects(dt, camera)   -- 特效更新
```

### core/CameraController.lua（相机控制）

**职责**：相机跟随、可视区域计算

```lua
-- 关键函数
CameraController.Init(scene, cameraNode)
CameraController.Update(dt, playerX, playerY)
CameraController.GetVisibleArea()  -- 返回 {minX, maxX, minY, maxY}
```

### core/InputHandler.lua（输入处理）

**职责**：键盘/触摸输入、UI交互分发

```lua
-- 状态

InputHandler.weaponSelectActive   -- 武器选择界面

-- 关键函数
InputHandler.Update(dt, gameState, bridgeState)
InputHandler.GetMovementInput()   -- 返回 moveX, moveY
```

### core/BridgeUpgrade.lua（舰桥升级）

**职责**：升级选项生成、选择、刷新

```lua

-- 关键函数
```

## 效果组件系统 ⭐

### 设计理念

效果组件系统将触发型效果（如"击杀回血"、"死亡连锁"）从硬编码改为数据驱动配置。

**之前（硬编码）**：
```lua
-- modules.lua 中设置标记
effect = function(p) p.killHeal = 1 end

-- Enemy.lua 中硬编码检查
if Game.player.killHeal > 0 then Game.Heal(killHeal) end
```

**现在（数据驱动）**：
```lua
-- config/effects.lua 中定义完整效果
Effects.Definitions["heal_on_kill"] = {
    trigger = Events.ENEMY_DEATH,
    condition = function(ctx) return ctx.player.killHeal > 0 end,
    execute = function(ctx) Game.Heal(ctx.player.killHeal) end,
}
```

### 效果定义结构

```lua
Effects.Definitions["effect_id"] = {
    trigger = Events.EVENT_NAME,     -- 触发事件
    condition = function(ctx) end,    -- 触发条件（可选）
    execute = function(ctx) end,      -- 执行逻辑
    priority = 0,                     -- 优先级（可选）
    description = "效果说明",          -- 描述（可选）
}
```

### 已实现效果

| 效果 ID | 触发事件 | 描述 |
|---------|---------|------|
| `heal_on_kill` | ENEMY_DEATH | 击杀回复护盾 |
| `demon_pact_heal` | ENEMY_DEATH | 恶魔契约击杀回血 |
| `death_chain` | ENEMY_DEATH | 死亡连锁伤害 |
| `kill_crystal_bonus` | ENEMY_DEATH | 击杀额外晶体 |
| `emergency_boost` | PLAYER_DAMAGE | 受伤加速 |
| `demon_pact_drain` | GAME_TICK | 每秒护盾消耗 |
| `plasma_burn` | WEAPON_HIT | 点燃效果 |

### 与模块系统协作

1. `modules.lua` 中的 `effect` 函数设置玩家属性标记
2. `effects.lua` 中的效果定义检测这些标记并执行实际逻辑
3. `EffectSystem.lua` 监听事件并自动触发匹配的效果

```
模块购买 → modules.effect() 设置标记 → 事件发生 → EffectSystem 检测标记 → 执行效果
```

## 模块通信模式

### 回调模式（当前主要方式）

模块通过回调函数通信，在 `main.lua` 的 `SetupCallbacks()` 中设置：

```lua
-- 示例：敌人死亡回调
Enemy.onDeath = function(enemy)
    Game.OnKill()
    Game.AddXp(enemy.dropXp)
    Pickup.CreateCrystal(scene_, ...)
    Effects.CreateDeathParticles(scene_, ...)
    Audio.PlayExplosion(enemy.isBoss)
end
```

### 事件总线（推荐用于新功能）

`utils/EventBus.lua` 提供发布/订阅模式，用于解耦：

```lua
local EventBus = require("utils.EventBus")
local Events = require("config.events")

-- 订阅
EventBus.on(Events.ENEMY_DEATH, function(eventData)
    -- 处理敌人死亡
end)

-- 发布
EventBus.emit(Events.ENEMY_DEATH, { enemy = enemy, damage = damage })
```

### 效果系统（数据驱动）

自动监听事件并执行匹配的效果：

```lua
-- 只需在 effects.lua 中定义效果
-- EffectSystem 会自动订阅相关事件并执行
```

## 敌人配置系统

### 核心字段说明

`config/enemies.lua` 中定义敌人属性：

| 字段 | 类型 | 说明 |
|------|------|------|
| `scale` | number | **统一尺寸**：视觉大小 = 碰撞半径 = 死亡粒子扩散范围 |
| `knockbackResist` | number | 击退抗性（0-1），0=完全击退，1=完全免疫 |
| `bodyColor` | table | 主体颜色 `{r, g, b}`（0-1 范围） |
| `glowColor` | table | 发光/爆炸颜色 `{r, g, b}`（0-1 范围） |
| `hp` | number | 基础生命值 |
| `damage` | number | 碰撞伤害 |
| `speed` | number | 移动速度 |
| `dropXp` | number | 击杀经验值 |
| `crystals` | table | 晶体掉落 `{min, max}` |

### scale 统一规则 ⭐

**重要设计**：`scale` 字段统一决定敌人的所有尺寸相关属性：

```
scale = 视觉大小 = 碰撞半径 = 死亡粒子扩散范围
```

- **视觉大小**：模型乘数已统一为 1.0，`scale` 直接等于视觉大小
- **碰撞半径**：`hitRadius = scale`（100% 一致，视觉命中即判定命中）
- **死亡粒子**：扩散范围基于 `scale` 计算，大型敌人爆炸更壮观

### 模型乘数规范

所有敌人模型（`entities/EnemyModels/*.lua`）的基准乘数为 **1.0**：

```lua
-- ✅ 正确：基准乘数为 1.0
local s = def.scale  -- s 直接是视觉大小
body:SetScale(Vector3(s * 1.0, s * 0.56, s * 0.5))

-- ❌ 错误：非 1.0 基准乘数
body:SetScale(Vector3(s * 1.6, s * 0.9, s * 0.8))  -- 会导致 scale 与视觉不一致
```

### knockbackResist 击退抗性

控制敌人被击中时的击退效果：

| 值 | 效果 |
|----|------|
| 0.0 | 完全击退，死亡粒子沿被击方向散射 |
| 0.5 | 50% 击退 |
| 1.0 | 完全免疫击退，死亡粒子随机散射 |

```lua
-- 示例配置
["Spore"] = {
    scale = 1.2,
    knockbackResist = 0,      -- 小型敌人，完全击退
    ...
},
["BroodMother"] = {
    scale = 2.4,
    knockbackResist = 0.8,    -- 大型敌人，高抗性
    ...
},
```

### glowColor 发光颜色

用于爆炸特效和发光材质，应与敌人类型视觉区分：

```lua
-- 孢子船：橙色
glowColor = {r = 1.0, g = 0.6, b = 0.2},

-- 炮舰：棕红色（避免与子弹混淆）
glowColor = {r = 0.9, g = 0.3, b = 0.2},

-- 自爆虫：红色
glowColor = {r = 1.0, g = 0.2, b = 0.1},
```

## 扩展指南

### 添加新敌人类型

1. 在 `config/enemies.lua` 添加敌人定义
   - `scale` 直接设置为期望的视觉大小（碰撞自动匹配）
   - `knockbackResist` 根据敌人体型设置（小型=0，大型=0.5-1.0）
   - `glowColor` 选择与其他敌人区分的颜色
2. 在 `entities/EnemyModels/` 添加模型文件
   - 基准乘数必须为 **1.0**
   - 其他部件相对于基准按比例设置
3. 在 `entities/EnemyModels/index.lua` 注册模型
4. 如有特殊行为，在 `Enemies.Behaviors` 添加行为类型

### 添加新武器

1. 在 `config/weapons.lua` 添加武器定义
2. 如有特殊攻击方式，在 `core/GameLoop.lua` 添加处理逻辑

### 添加新模块（道具）

1. 在 `config/modules.lua` 添加模块定义
2. 定义 `effect` 函数修改玩家属性
3. **如有触发型效果**，在 `config/effects.lua` 添加效果定义

### 添加新触发型效果 ⭐

只需在 `config/effects.lua` 中添加效果定义：

```lua
Effects.Definitions["new_effect"] = {
    trigger = Events.SOME_EVENT,
    condition = function(ctx)
        return ctx.player.someProperty > 0
    end,
    execute = function(ctx)
        -- 执行效果逻辑
    end,
}
```

无需修改其他代码，EffectSystem 会自动识别并处理。

## 性能注意事项

1. **延迟加载**：`GameLoop.lua` 等模块使用 `LoadDependencies()` 延迟加载避免循环依赖
2. **对象池**：特效和子弹使用列表管理，及时清理
3. **配置缓存**：配置模块在加载时预处理，避免运行时计算
4. **事件优化**：EffectSystem 只订阅有效果监听的事件
5. **条件检查**：效果的 `condition` 函数应快速返回
