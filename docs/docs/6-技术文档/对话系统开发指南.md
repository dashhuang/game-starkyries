# 对话系统开发文档

> 星河战姬 Starkyries - 剧情模式对话系统

## 概述

对话系统用于展示游戏剧情，支持：
- 多角色对话
- 场景图片切换
- 打字机效果
- 简单特效（如震动）
- 跳过功能（显示剧情简介）
- 图片占位符（开发期间使用）

**设计原则**：简单、直接，不支持分支选项。

---

## 核心设计规则

### 规则 #1: 指挥官不说话

玩家扮演的"指挥官"是**沉默型主角**，在对话中不发言。

```lua
-- ❌ 错误
{ speaker = "指挥官", text = "我明白了" }

-- ✅ 正确：角色解读指挥官的沉默
{ speaker = "星遥", text = "指挥官，你觉得呢？......嗯，我懂了！" }
{ speaker = "星遥", text = "......嗯，谢谢你不怪我。" }
```

**角色可以：**
- 直接对指挥官说话（称呼"指挥官"）
- 解读指挥官的沉默/表情/点头作为回应
- 通过对话选项让玩家参与（未来功能）

**设计原因：**
- 增强玩家代入感
- 避免替玩家说出玩家不认同的话
- 符合日式 RPG 经典设计

---

### 规则 #2: 图片场景一致性

所有角色图片都**自带背景场景**，同一段对话中不能随意切换场景。

```
❌ 穿帮示例：
对话1: 星遥/舰桥_开心.jpg    (舰桥背景)
对话2: 星遥/战斗_紧张.jpg    (战斗背景) ← 背景突然变了！

✅ 正确示例：
对话1: 星遥/战败_沮丧.jpg    (战败舰桥背景)
对话2: 星遥/战败_开心.jpg    (战败舰桥背景) ← 同一场景
```

**设计原则：**
- 每段对话开始时确定场景
- 该场景内的所有图片使用相同背景
- 切换场景 = 切换对话段落
- 不同角色可以有不同的背景（如通讯画面切换）

---

### 规则 #3: 图片命名规范

**路径格式：** `image/[角色名]/[场景_情绪].jpg`

**命名原则：**
| 规则 | 说明 | 示例 |
|------|------|------|
| 默认场景 | 舰桥是默认场景，无需前缀 | `冷静.jpg` |
| 特殊场景 | 需要标注场景名 | `战败_沮丧.jpg` |
| 情绪词 | 简洁明了 | 沮丧、开心、冷静、疑惑、微笑 |

**图片数量控制：**
- 每段对话每个角色控制在 **2-3 张**图片
- 优先考虑复用性和通用性
- 避免为单一对话创建过多专用图片

## 快速开始

```lua
local DialogueUI = require("ui.DialogueUI")
local DialogueData = require("data.DialogueData")

-- 方式1：使用预定义对话
DialogueUI.Play(DialogueData.Tutorial_FirstDefeat, function()
    print("对话结束")
end)

-- 方式2：直接传入对话数据
DialogueUI.Play({
    summary = "剧情简介，跳过时显示",
    { speaker = "星遥", text = "指挥官，准备好了吗？", image = "image/星遥/开心.jpg" },
    { speaker = "星遥", text = "那我们出发吧！" },
}, function()
    StartBattle()
end)
```

## 对话数据格式

### 对话序列字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `summary` | string | 否 | 剧情简介，玩家跳过对话时显示 |

### 对话条目字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `speaker` | string | 是 | - | 说话者名称，显示在对话框上方 |
| `text` | string | 是 | - | 对话文本，支持中英文混合 |
| `image` | string | 否 | 保持上一张 | 角色图片路径，相对于 assets 目录 |
| `imageDesc` | string | 否 | - | 图片描述，用于占位符显示 |
| `cropTop` | number | 否 | 0.15 | 横屏时图片顶部裁剪比例（0-1） |
| `effect` | string | 否 | 无 | 触发效果，见下方效果列表 |

### 可用效果

| 效果名 | 说明 |
|--------|------|
| `shake` | 画面震动，持续0.3秒 |

## 跳过功能

### 功能说明

玩家可以跳过对话，跳过时显示剧情简介让玩家了解关键剧情。

**操作方式：**
- 点击右上角"跳过"按钮
- 按 ESC 键

**跳过后：**
- 显示剧情简介界面
- 点击任意位置继续

### summary 字段

在对话数据中添加 `summary` 字段定义剧情简介：

```lua
DialogueData.Tutorial_FirstDefeat = {
    -- 剧情简介（跳过时显示）
    summary = "首次战斗失利后，星遥联系了工程专家零请求支援。零决定驾驶赛博号与星遥并肩作战。",
    
    -- 对话条目
    { speaker = "星遥", text = "..." },
    ...
}
```

## 图片占位符系统

创作对话时可以引用**尚不存在的图片**，系统会自动显示占位符。

**占位符显示内容：**
- 灰色网格背景
- 图片路径
- 图片描述（来自 `imageDesc` 字段）

**使用方法：**
```lua
{
    speaker = "星遥",
    text = "对话内容",
    image = "image/星遥/战败_沮丧.jpg",
    imageDesc = "星遥半身像，表情沮丧低落，先锋号舰桥背景",
}
```

**工作流程：**
1. 编剧创作对话，标注所需图片路径和描述
2. 测试时看到占位符，确认对话流程
3. 美术根据 `imageDesc` 制作图片
4. 图片放入对应目录，占位符自动替换，可删除 `imageDesc`

## 图片显示规则

### Cover 模式

图片始终填满屏幕，根据屏幕方向自动裁剪：

- **横屏**：裁剪上下，默认裁掉顶部 `cropTop`（15%）的多余区域
- **竖屏**：垂直居中显示

### 图片规格建议

- **推荐比例**：1:1 方形
- **推荐尺寸**：1024x1024 或 2048x2048
- **格式**：JPG（较小体积）

### cropTop 参数说明

```
cropTop = 0      → 顶部对齐（保留头部）
cropTop = 0.15   → 默认值，裁掉顶部15%多余区域
cropTop = 0.40   → 裁掉顶部40%（适合全身像）
cropTop = 0.5    → 居中对齐
```

## 完整案例：首次战败

### 场景背景

- **触发时机**：新手引导，玩家首次战斗失败后
- **参与角色**：星遥（先锋号舰长）、零（赛博号舰长）
- **剧情目的**：引出零加入战斗，解锁零为可操控角色

### 图片资源

```
assets/image/
├── 星遥/
│   ├── 战败_沮丧.jpg    # cropTop = 0.40
│   └── 战败_开心.jpg    # cropTop = 0.40
└── 零/
    ├── 冷静.jpg         # 默认 cropTop
    ├── 疑惑.jpg         # 默认 cropTop
    └── 微笑.jpg         # 默认 cropTop
```

### 对话结构

```
第一幕：战败后（星遥沮丧）
├── 星遥道歉，表达沮丧
├── 指挥官沉默安慰（通过星遥的反应体现）
└── 星遥振作，提议找零帮忙

第二幕：联系零
├── 星遥呼叫零
└── 零接通通讯

第三幕：零分析战况
├── 零分析受损情况
├── 零认出指挥官（旧识）
└── 零同意支援

第四幕：引出零参战
├── 零分析单舰胜率低
├── 星遥沮丧
├── 零提议协同作战
├── 零表示想报答指挥官
└── 解锁零为可操控角色
```

### 设计要点

1. **指挥官不说话**：通过"......嗯，谢谢你不怪我"让星遥解读指挥官的沉默
2. **图片场景一致**：星遥始终用"战败"场景图片，零始终用默认舰桥图片
3. **角色性格体现**：
   - 星遥：活泼、情绪外露、称呼亲切
   - 零：冷静、数据化表达、偶尔流露感情
4. **剧情目的达成**：自然引出零加入战斗，玩家解锁新角色
5. **图片复用性**：只用 5 张图片完成整段对话

## API 参考

### DialogueUI.Play(dialogues, onComplete)

播放对话序列。

```lua
---@param dialogues table 对话数据数组（可包含 summary 字段）
---@param onComplete function? 完成回调（可选）
DialogueUI.Play(dialogues, onComplete)
```

### DialogueUI.SkipDialogue()

跳过对话，显示剧情简介（如果有）。

```lua
DialogueUI.SkipDialogue()
```

### DialogueUI.Stop()

立即停止对话（不显示简介）。

```lua
DialogueUI.Stop()
```

### DialogueUI.IsVisible()

检查是否正在显示对话。

```lua
if DialogueUI.IsVisible() then
    -- 对话正在进行
end
```

### DialogueUI.IsTyping()

检查是否正在打字（文字还在逐字显示）。

```lua
if DialogueUI.IsTyping() then
    -- 文字正在显示中
end
```

## 配置项

在 `DialogueUI.Config` 中可调整：

```lua
DialogueUI.Config = {
    -- 打字机效果
    charDelay = 0.03,           -- 每个字符的延迟（秒）
    fastCharDelay = 0.005,      -- 快进时的延迟
    
    -- 图片裁剪
    defaultCropTop = 0.15,      -- 默认裁掉顶部15%
    
    -- 对话框样式
    boxHeightRatio = 0.28,      -- 对话框高度占屏幕比例
    boxMargin = 2,              -- 对话框边距（baseUnit倍数）
    
    -- 效果
    shakeDuration = 0.3,        -- 震动持续时间
    shakeIntensity = 8,         -- 震动强度（像素）
}
```

## 注意事项

1. **指挥官不说话**：这是核心设计规则，所有对话都必须遵守
2. **图片场景一致**：同一段对话中角色图片背景必须一致
3. **图片预加载**：图片在首次显示时才加载，场景切换时调用 `DialogueUI.ClearImageCache()` 清理
4. **UTF-8支持**：对话文本完全支持中文，打字机效果按字符显示
5. **输入阻断**：对话显示时会阻断其他游戏输入
6. **跳过简介**：重要剧情必须提供 `summary` 字段，让跳过的玩家也能了解剧情

## 配音系统

### 概述

对话系统支持角色配音，使用 ElevenLabs TTS API 生成语音文件。

**相关文档**：
- 技术详情 → `tools/README_VOICE.md`
- AI Skill → `.claude/skills/voice.md`

### 配音字段

在对话条目中添加 `emotion` 字段可以控制配音效果：

```lua
{
    speaker = "星遥",
    text = "指挥官，我们一定能赢！",
    image = "image/星遥/坚定.jpg",
    emotion = "坚定",  -- 配音时自动应用情绪效果
}
```

### 已配置角色

| 角色 | 状态 | 说明 |
|------|------|------|
| 星遥 | 已配置 | 活泼开朗 |
| 零 | 已配置 | 冷静理性 |
| 蚀 | 已配置 | 冷漠疏离 |
| 伊芙 | 已配置 | 神秘多变，Arabella 声线 |
| 其他 | 待配置 | 详见 README_VOICE.md |

### 生成配音

创建对话后生成配音：

```bash
# 生成所有配音（API Key 从 .env 读取）
python tools/voice_generator.py

# 只处理特定文件
python tools/voice_generator.py --file scripts/data/dialogue/daily/BridgeChat.lua

# 重新生成某一句
python tools/voice_generator.py --file <文件> --index 5 --force
```

### 情绪映射

| 情绪 | 效果 | 适用场景 |
|------|------|----------|
| 激昂 | 热血激动 | 出击、战斗 |
| 紧张 | 紧急慌张 | 危险、警报 |
| 开心 | 快乐欢喜 | 胜利、好消息 |
| 坚定 | 决心果断 | 鼓励、决定 |
| 温柔 | 柔和关怀 | 安慰、日常 |
| 冷静 | 平静理性 | 分析、报告 |

**完整映射表** → `tools/README_VOICE.md`

### 工作流程

```
1. 创建对话脚本（添加 emotion 字段）
2. 运行 python tools/voice_generator.py
3. 未配置语音的角色会被跳过
4. 构建测试 → build
```
