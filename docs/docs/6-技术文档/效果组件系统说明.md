# 星河战姬 - 效果组件系统（Effect Component System）

> **版本**：v1.0
> **创建日期**：2026-01-02
> **架构类型**：混合架构（OOP + 数据驱动效果）

---

## 概述

效果组件系统是为支持复杂道具和角色技能而设计的数据驱动架构。它保留了现有的 OOP 结构，同时引入了组件化的效果处理机制。

### 设计动机

当前 `modules.lua` 中的效果系统存在局限：

```lua
-- 当前方式：只能处理被动属性加成
{
    id = "kill_thrill",
    effect = function(p)
        p.killHeal = (p.killHeal or 0) + 1  -- 只是设置标记
    end,
}
-- 实际效果需要在其他地方硬编码：
-- Enemy.lua: if Game.player.killHeal then Game.Heal(killHeal) end
```

**问题**：
1. 效果逻辑分散在多个文件中
2. 添加新效果需要修改多个模块
3. 效果之间难以组合和扩展
4. 条件判断和触发时机硬编码

### 解决方案

效果组件系统将效果定义为独立的数据单元：

```lua
-- 新方式：效果完全自描述
Effects.Definitions["heal_on_kill"] = {
    trigger = Events.ENEMY_DEATH,      -- 触发时机
    condition = function(ctx)          -- 触发条件
        return ctx.player.killHeal > 0
    end,
    execute = function(ctx)            -- 执行逻辑
        Game.Heal(ctx.player.killHeal)
    end,
}
```

## 效果定义规范

### 基本结构

```lua
-- config/effects.lua
Effects.Definitions["effect_id"] = {
    -- 必填：触发事件
    trigger = Events.EVENT_NAME,
    
    -- 可选：触发条件（返回 true 时执行）
    condition = function(ctx) 
        return true 
    end,
    
    -- 必填：执行逻辑
    execute = function(ctx)
        -- 效果实现
    end,
    
    -- 可选：优先级（数值越大越先执行，默认 0）
    priority = 0,
    
    -- 可选：是否可叠加执行（默认 true）
    stackable = true,
    
    -- 可选：描述（用于 UI 显示）
    description = "效果说明",
}
```

### 上下文对象（ctx）

效果执行时会收到上下文对象，包含当前游戏状态：

```lua
ctx = {
    -- 玩家状态（只读建议）
    player = Game.player,
    
    -- 事件参数（根据触发事件不同）
    event = {
        -- ENEMY_DEATH: enemy, damage, isCrit, position
        -- PLAYER_DAMAGE: damage, finalDamage, blocked
        -- WEAPON_HIT: weapon, target, damage, isCrit
        -- ...
    },
    
    -- 效果元数据
    effectId = "effect_id",
    stackCount = 1,  -- 该效果的叠加层数
    
    -- 工具函数
    emit = function(event, ...)  -- 发射新事件
        EventBus.emit(event, ...)
    end,
}
```

### 效果类型分类

#### 1. 击杀触发效果

```lua
-- 击杀回血
Effects.Definitions["heal_on_kill"] = {
    trigger = Events.ENEMY_DEATH,
    condition = function(ctx)
        return ctx.player.killHeal and ctx.player.killHeal > 0
    end,
    execute = function(ctx)
        local healAmount = ctx.player.killHeal * ctx.stackCount
        Game.Heal(healAmount)
    end,
}

-- 死亡连锁
Effects.Definitions["death_chain"] = {
    trigger = Events.ENEMY_DEATH,
    condition = function(ctx)
        return ctx.player.deathChainChance and 
               math.random() < ctx.player.deathChainChance
    end,
    execute = function(ctx)
        local enemy = ctx.event.enemy
        local chainDamage = ctx.event.damage * 0.5
        -- 对附近敌人造成连锁伤害
        EffectSystem.TriggerChainDamage(enemy.x, enemy.y, chainDamage, 80)
    end,
}
```

#### 2. 受伤触发效果

```lua
-- 紧急加速
Effects.Definitions["emergency_boost"] = {
    trigger = Events.PLAYER_DAMAGE,
    condition = function(ctx)
        return ctx.player.hasEmergencyBoost and
               not ctx.player.emergencyBoostActive
    end,
    execute = function(ctx)
        ctx.player.emergencyBoostActive = true
        ctx.player.emergencyBoostTimer = ctx.player.emergencyBoostDuration
        -- 临时速度加成由 Player.lua 的 Update 处理
    end,
}

-- 恶魔契约（击杀回血增强）
Effects.Definitions["demon_pact_heal"] = {
    trigger = Events.ENEMY_DEATH,
    condition = function(ctx)
        return ctx.player.hasDemonPact
    end,
    execute = function(ctx)
        Game.Heal(ctx.player.demonPactKillHeal)
    end,
    priority = 10,  -- 优先于普通击杀回血
}
```

#### 3. 条件性伤害加成

```lua
-- 满盾伤害加成（被动计算，非触发型）
-- 这类效果在伤害计算时检查，不通过 EffectSystem
-- 保留在原有的 damageMultiplier 计算逻辑中
```

#### 4. 周期性效果

```lua
-- 恶魔契约（每秒扣血）
Effects.Definitions["demon_pact_drain"] = {
    trigger = Events.GAME_TICK,  -- 每秒触发一次
    condition = function(ctx)
        return ctx.player.hasDemonPact
    end,
    execute = function(ctx)
        Game.TakeDamage(ctx.player.demonPactDrain, true)  -- true = 无敌时间不生效
    end,
}
```

## EffectSystem API

### 初始化

```lua
local EffectSystem = require("core.EffectSystem")

function Start()
    -- 初始化效果系统（自动订阅所有效果的触发事件）
    EffectSystem.Init()
end
```

### 核心函数

```lua
-- 初始化效果系统
EffectSystem.Init()

-- 手动触发效果检查（通常由事件自动触发）
EffectSystem.CheckTrigger(eventName, eventData)

-- 注册新效果（运行时动态添加）
EffectSystem.Register(effectId, definition)

-- 移除效果
EffectSystem.Unregister(effectId)

-- 获取玩家当前激活的效果列表
EffectSystem.GetActiveEffects()

-- 获取效果的叠加层数
EffectSystem.GetStackCount(effectId)

-- 调试：打印效果系统状态
EffectSystem.Debug()
```

### 特殊触发函数

```lua
-- 连锁伤害（供 death_chain 等效果调用）
EffectSystem.TriggerChainDamage(x, y, damage, radius)

-- AOE 伤害
EffectSystem.TriggerAOEDamage(x, y, damage, radius, excludeEnemy)

-- 屏幕特效
EffectSystem.TriggerScreenEffect(effectType, params)
```

## 与现有系统的集成

### 模块系统集成

`modules.lua` 中的模块仍然使用 `effect` 函数设置玩家属性标记：

```lua
-- modules.lua（保持不变）
{
    id = "kill_thrill",
    effect = function(p)
        p.killHeal = (p.killHeal or 0) + 1  -- 设置标记
    end,
}
```

EffectSystem 负责检测这些标记并执行实际效果：

```lua
-- effects.lua
Effects.Definitions["heal_on_kill"] = {
    trigger = Events.ENEMY_DEATH,
    condition = function(ctx)
        return ctx.player.killHeal and ctx.player.killHeal > 0
    end,
    execute = function(ctx)
        Game.Heal(ctx.player.killHeal)
    end,
}
```

### 事件发射点

需要在关键位置发射事件（已有回调的地方改为发射事件）：

```lua
-- entities/Enemy.lua
function Enemy.Kill(enemy, damage, isCrit)
    -- 原有逻辑...
    
    -- 发射事件（EffectSystem 会自动响应）
    EventBus.emit(Events.ENEMY_DEATH, {
        enemy = enemy,
        damage = damage,
        isCrit = isCrit,
        position = {x = enemy.x, y = enemy.y},
    })
end
```

### 渐进式迁移

效果组件系统支持渐进式迁移：

1. **阶段 1**：添加 EffectSystem 基础设施
2. **阶段 2**：为新效果使用 EffectSystem
3. **阶段 3**：逐步迁移现有硬编码效果
4. **阶段 4**：统一所有效果通过 EffectSystem 管理

## 使用示例

### 添加新效果（完整流程）

1. **定义效果**（config/effects.lua）：

```lua
Effects.Definitions["vampire_rounds"] = {
    trigger = Events.WEAPON_HIT,
    condition = function(ctx)
        return ctx.player.vampireRounds and ctx.player.vampireRounds > 0
    end,
    execute = function(ctx)
        local healAmount = ctx.event.damage * 0.05 * ctx.player.vampireRounds
        Game.Heal(math.floor(healAmount))
    end,
    description = "攻击时回复造成伤害的5%护盾",
}
```

2. **添加模块**（config/modules.lua）：

```lua
{
    id = "vampire_rounds",
    name = "吸血弹",
    type = Modules.Types.TACTICAL,
    description = "攻击回复5%伤害为护盾",
    price = 45,
    maxStack = 3,
    rarity = 2,
    effect = function(p)
        p.vampireRounds = (p.vampireRounds or 0) + 1
    end,
},
```

3. **确保事件发射**（如果尚未添加）：

```lua
-- core/GameLoop.lua 的武器命中处理中
EventBus.emit(Events.WEAPON_HIT, {
    weapon = weapon,
    target = enemy,
    damage = damage,
    isCrit = isCrit,
})
```

完成！无需修改其他代码。

## 性能考虑

### 事件订阅优化

EffectSystem 只订阅有效果监听的事件：

```lua
function EffectSystem.Init()
    -- 收集所有效果使用的触发事件
    local usedTriggers = {}
    for _, def in pairs(Effects.Definitions) do
        usedTriggers[def.trigger] = true
    end
    
    -- 只订阅使用中的事件
    for trigger in pairs(usedTriggers) do
        EventBus.on(trigger, function(eventData)
            EffectSystem.CheckTrigger(trigger, eventData)
        end)
    end
end
```

### 条件检查优化

效果的 `condition` 函数应该快速返回：

```lua
-- ✅ 好：简单属性检查
condition = function(ctx)
    return ctx.player.killHeal > 0
end

-- ❌ 差：复杂计算放在 condition 中
condition = function(ctx)
    local nearbyEnemies = Enemy.GetInRadius(ctx.event.x, ctx.event.y, 100)
    return #nearbyEnemies > 3  -- 每次事件都要遍历敌人
end
```

### 叠加效果处理

对于可叠加效果，`stackCount` 已预计算：

```lua
execute = function(ctx)
    -- ctx.stackCount 已包含叠加层数
    local totalHeal = ctx.player.killHeal * ctx.stackCount
    Game.Heal(totalHeal)
end
```

### 效果持续时间

```lua
Effects.Definitions["temporary_shield"] = {
    trigger = Events.PICKUP_COLLECT,
    condition = function(ctx) return ctx.event.pickup.type == "shield_boost" end,
    execute = function(ctx)
        ctx.player.tempShieldBonus = 20
    end,
    duration = 10.0,  -- 10秒后自动移除
    onExpire = function(ctx)
        ctx.player.tempShieldBonus = 0
    end,
}
```

## 总结

效果组件系统提供了：

| 特性 | 描述 |
|------|------|
| **数据驱动** | 效果通过配置定义，无需修改核心代码 |
| **解耦设计** | 效果逻辑集中管理，与游戏系统分离 |
| **可扩展性** | 添加新效果只需在 effects.lua 中定义 |
| **向后兼容** | 现有模块系统继续工作，渐进式迁移 |
| **调试友好** | 内置调试模式和统计功能 |

这种混合架构既保留了 OOP 的清晰结构，又获得了组件系统的灵活性，非常适合支持复杂的道具和技能系统。
