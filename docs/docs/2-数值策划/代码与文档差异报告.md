## 一、核心问题总结

### 🔴 严重问题

| 问题 | 说明 | 影响 |
|------|------|------|
| **代码缺少代价机制** | 文档要求所有正面效果必须有对应代价，但代码中绝大部分模块没有实现代价 | 破坏游戏平衡，与Brotato设计理念背离 |
| **能量吸收数值差异巨大** | 代码：+15% / 文档：+3% | 代码数值是文档的5倍，严重破坏平衡 |
| **类型加成数值过高** | 代码：+20% / 文档：+15% | 叠加后差异更大 |

### 🟡 中等问题

| 问题 | 说明 |
|------|------|
| 部分模块ID重复 | `ballistic_computer` 出现两次 |
| 文档有模块代码未实现 | 约120+模块仅文档定义 |
| 代码有模块文档未提及 | 如"战场直觉"、"银弹"、"经验强化器" |

## 四、修复建议

### 优先级 1：修复现有模块数值（对齐文档）

```lua
-- 示例：装甲板修复
{
    id = "armor_plate",
    name = "装甲板",
    description = "装甲+2，火力输出-3%",
    price = 40,  -- 原25改为40
    effect = function(p)
        p.armor = p.armor + 2  -- 原+3改为+2
        p.damageMultiplier = p.damageMultiplier - 0.03  -- 新增代价
    end,
}

-- 示例：能量虹吸核心修复
{
    id = "energy_siphon",
    name = "虹吸核心",
    description = "能量吸收+3%，最大护盾-5",
    price = 35,  -- 原30改为35
    effect = function(p)
        p.energyAbsorb = p.energyAbsorb + 0.03  -- 原0.15改为0.03
        p.maxShield = p.maxShield - 5  -- 新增代价
    end,
}
```

### 优先级 2：实现代价机制

所有模块必须实现对应代价，建议在 effect 函数中统一处理：

```lua
effect = function(p)
    -- 正面效果
    p.armor = p.armor + 2
    -- 代价
    p.damageMultiplier = p.damageMultiplier - 0.03
end,
```

### 优先级 3：清理重复ID

`ballistic_computer` 出现两次，需要修复：
- 保留通用火力加成版本
- 或改为类型专精版本（仅加成弹道武器）

### 优先级 4：补充缺失模块

按游戏进度需要，逐步实现文档中定义的其他模块。

## 五、总结

| 统计项 | 数量 |
|--------|------|
| 代码实现模块数 | 25 |
| 文档定义模块数 | ~144 |
| 数值一致模块 | 3 (实验模块) |
| 需修复数值模块 | 22 |
| 缺少代价模块 | 22 |
| 代码独有模块 | 3 |

**核心结论**：代码与文档严重不一致，需要大规模同步更新。建议优先修复现有模块的数值和代价，再逐步补充缺失模块。
