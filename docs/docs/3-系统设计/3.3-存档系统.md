# 3.3 存档系统

> 即时存档、玩家档案、数据结构、云同步设计

---

## 概述

存档系统分为两大部分：

| 类型 | 说明 | 生命周期 | 存储方式 |
|------|------|----------|----------|
| **单局存档** | 当前游戏进度（波次、战舰状态等） | 单局游戏内 | 本地优先 + 云同步 |
| **玩家档案** | 永久进度（成就、解锁、生涯统计等） | 永久保存 | 云端存储 |

---

# 第一部分：单局存档

> 记录当前游戏局的进度，支持中途退出后继续

## 1. 存档时机

| 时机 | 存档点 | 云同步 | 说明 |
|------|--------|--------|------|
| **波次开始前** | ✅ 自动存档 | ✅ 同步 | 进入波次前的完整状态 |
| **波次开始后** | ✅ 自动存档 | ✅ 同步 | 标记"波次进行中" |
| **波次结束后** | ✅ 自动存档 | ✅ 同步 | 升级/补给站前的状态 |
| **补给站操作后** | ✅ 自动存档 | ❌ 仅本地 | 购买/刷新后更新 |
| **退出补给站** | ✅ 自动存档 | ✅ 同步 | 确保补给站操作持久化 |

---

## 2. 中途离开处理

```
玩家在波次中离开游戏
        ↓
再次启动游戏 → 点击"继续游戏"
        ↓
┌─────────────────────────────────────────┐
│  检测到未完成的波次 (第 N 波)            │
│                                         │
│  上次进度将回退到波次开始前的状态        │
│                                         │
│  [重新开始波次]     [返回主界面]         │
└─────────────────────────────────────────┘
        ↓
点击"重新开始波次" → 从波次开始前的存档恢复
```

| 场景 | 恢复状态 | 说明 |
|------|----------|------|
| **波次进行中离开** | 回到波次开始前 | 波次内获得的经验/晶体不保留 |
| **补给站中离开** | 回到补给站 | 保留当前购买状态 |
| **升级选择中离开** | 回到升级界面 | 保留已选择的升级 |

---

## 3. 单局存档内容

| 数据类型 | 包含内容 |
|----------|----------|
| **进度** | 当前波次、波次状态（进行中/已完成） |
| **战舰** | 当前护盾、最大护盾、所有属性 |
| **装备** | 已装备武器（含Tier）、已装备模块 |
| **经济** | 能量晶体数量 |
| **升级** | 舰长等级、已选择的属性升级 |
| **统计** | 本局击杀数、获得晶体总数等 |

---

## 4. 单局存档规则

| 规则 | 说明 |
|------|------|
| **存档数量** | 每个账号 1 个即时存档 |
| **按钮显示** | 仅当存在有效存档时显示"继续游戏" |
| **存档覆盖** | 开始新游戏时提示"将覆盖现有存档" |
| **通关清除** | 第20波胜利或战舰损毁后，清除即时存档 |

---

## 5. 防作弊设计

| 机制 | 效果 |
|------|------|
| **波次回退** | 波次中退出 = 放弃本波次进度，防止"快死了就退出" |
| **无法手动存档** | 只能自动存档，无法选择存档点 |
| **单存档槽** | 无法保留多个进度状态 |

# 第二部分：玩家档案

> 永久保存的玩家数据，跨局游戏累积

## 6. 玩家档案概述

玩家档案记录所有**跨局永久保存**的数据：

| 数据分类 | 说明 |
|----------|------|
| **生涯统计** | 累计游戏数据（击杀、通关次数等） |
| **成就系统** | 成就解锁状态和进度 |
| **舰长数据** | 解锁状态、好感度、剧情进度 |
| **图鉴数据** | 武器/模块/敌人解锁状态 |
| **解锁数据** | 战舰解锁、皮肤解锁等 |
| **设置数据** | 音量、语言等用户偏好 |

---

## 8. 成就系统数据

### 8.1 成就分类

| 分类 | 示例成就 | 说明 |
|------|----------|------|
| **战斗成就** | 击杀10000敌舰、单局击杀500敌舰 | 战斗相关 |
| **通关成就** | 首次通关、危险5通关 | 通关相关 |
| **收集成就** | 解锁所有武器、解锁所有舰长 | 收集相关 |
| **挑战成就** | 无伤通关、单武器通关 | 特殊挑战 |
| **舰长成就** | 某舰长好感度满级 | 养成相关 |
| **隐藏成就** | 特殊条件触发 | 惊喜发现 |

### 8.2 成就数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `achievements` | map | 成就数据 |
| └─ `{achievementId}.unlocked` | bool | 是否已解锁 |
| └─ `{achievementId}.unlockTime` | int | 解锁时间戳 |
| └─ `{achievementId}.progress` | int | 当前进度（未解锁时） |
| └─ `{achievementId}.target` | int | 目标值 |

### 8.3 成就示例定义

| ID | 名称 | 条件 | 奖励 |
|----|------|------|------|
| `kill_1000` | 新手猎人 | 累计击杀1000敌舰 | 10晶体 |
| `kill_10000` | 王牌猎人 | 累计击杀10000敌舰 | 50晶体 |
| `kill_100000` | 传奇猎人 | 累计击杀100000敌舰 | 200晶体 |
| `first_clear` | 初次凯旋 | 首次通关 | 解锁新战舰 |
| `danger5_clear` | 极限挑战 | 危险5通关 | 专属称号 |
| `no_damage_clear` | 完美战役 | 无伤通关 | 专属皮肤 |
| `single_weapon` | 独行侠 | 单武器通关 | 成就徽章 |
| `unlock_all_weapons` | 武器大师 | 解锁所有武器图鉴 | 专属称号 |
| `captain_max_affection` | 誓约之证 | 任意舰长好感度满级 | 誓约戒指 |

# 第三部分：存储与同步

## 13. 存储策略

### 13.1 本地优先 + 云同步

```
┌─────────────────────────────────────────────────────┐
│                    存储架构                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│   ┌─────────────┐     ┌─────────────┐              │
│   │  内存缓存   │ ←→  │  云端存储   │              │
│   │  (快速)     │     │ (clientScore)│              │
│   └─────────────┘     └─────────────┘              │
│         ↑                   ↑                       │
│         │                   │                       │
│   频繁操作仅存内存      关键节点同步云端            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 13.2 同步时机

| 数据类型 | 同步时机 | 说明 |
|----------|----------|------|
| **单局存档** | 波次开始/结束、退出补给站 | 关键节点同步 |
| **生涯统计** | 游戏结束时 | 一局结束后同步 |
| **成就解锁** | 解锁瞬间 | 立即同步，防丢失 |
| **舰长好感度** | 游戏结束时 | 一局结束后同步 |
| **图鉴解锁** | 首次发现时 | 立即同步 |
| **设置** | 修改后3秒 | 防抖动后同步 |

### 13.3 冲突处理

| 场景 | 处理策略 |
|------|----------|
| 云端数据更新 | 云端优先（累加型数据取最大值） |
| 成就解锁 | 已解锁状态不可撤销 |
| 好感度 | 取最大值 |
| 统计数据 | 取最大值 |

## 15. API 设计

### 15.1 存档管理器接口

```lua
-- 单局存档
SaveManager.SaveSession(Game, waveState)    -- 保存单局存档
SaveManager.LoadSession() -> SessionSave    -- 加载单局存档
SaveManager.ClearSession()                   -- 清除单局存档
SaveManager.HasSession() -> bool             -- 是否有存档

-- 玩家档案
ProfileManager.GetProfile() -> PlayerProfile -- 获取玩家档案
ProfileManager.SaveProfile()                 -- 保存玩家档案

-- 生涯统计
ProfileManager.AddKills(count)               -- 增加击杀数
ProfileManager.AddGameResult(won, wave)      -- 记录游戏结果
ProfileManager.AddPlayTime(seconds)          -- 增加游戏时长

-- 成就
AchievementManager.Check(achievementId)      -- 检查成就条件
AchievementManager.Unlock(achievementId)     -- 解锁成就
AchievementManager.UpdateProgress(id, value) -- 更新进度

-- 舰长
CaptainManager.AddAffection(captainId, points) -- 增加好感度
CaptainManager.UnlockStory(captainId, chapter) -- 解锁剧情
CaptainManager.Oath(captainId)                 -- 誓约

-- 图鉴
CollectionManager.DiscoverWeapon(weaponId, tier) -- 发现武器
CollectionManager.DiscoverModule(moduleId)       -- 发现模块
CollectionManager.DiscoverEnemy(enemyId)         -- 发现敌人
```

## 16. 云存储 Key 设计

使用 `clientScore` API 存储，Key 规划：

| Key | 内容 | 说明 |
|-----|------|------|
| `session_save` | 单局存档JSON | 单局游戏进度 |

| `profile_career` | 生涯统计JSON | 累计数据 |
| `profile_achievements` | 成就数据JSON | 成就解锁状态 |
| `profile_captains` | 舰长数据JSON | 舰长养成进度 |
| `profile_collections` | 图鉴数据JSON | 武器/模块/敌人图鉴 |
| `profile_ships` | 战舰数据JSON | 战舰解锁状态 |
| `profile_settings` | 设置数据JSON | 用户偏好 |
