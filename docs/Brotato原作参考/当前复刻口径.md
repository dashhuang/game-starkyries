## 道具数量口径

这里不能简单写死为一个数字：

- Wiki `Items` 页面正文声明：201个原版道具 + 36个Abyssal Terrors DLC道具 = 237个。
- 当前 `Template:Item Data` 原始模板解析出239个条目。
- 当前 `Items` 主表通过HTML表格解析为231行。

模板中额外出现但未进入Items主表的条目包括：`ban system`、`broken hourglass`、`broken mirror`、`builder's turret`、`builder's turret ii`、`builder's turret iii`、`builder's turret iv`、`resting goldfish` 等。它们更像内部、衍生、角色机制或特殊状态条目；在没有进一步证据前，不能直接当作普通可购道具处理。

本地数据库仍以官方页面正文的237个道具口径描述“可收集/可购道具总数”，同时在快照中保留239个模板条目，供复刻时处理内部实体和特殊状态。

## 当前商店刷新公式

以 `数据快照/2026-05-11/raw_html/Shop.html` 页面顶部公式为准：

```text
Reroll Increase = Rounddown(0.40 * Wave Number), minimum 1
First Reroll Price = Rounddown(Wave Number * 0.75) + Reroll Increase
Nth Reroll Price = First Reroll Price + (N - 1) * Reroll Increase
```

注意：Shop页面底部补丁历史仍提到1.1.4.0曾改为 `0.33 * Wave` / `Wave / 2`，但这与当前页面顶部现行公式冲突。当前复刻口径采用顶部现行公式，并保留历史差异说明。

## 不能假装已经完整的部分

- 每波精确刷怪脚本：敌人组、数量曲线、生成间隔、上限处理、组权重、地图/模式差异。
- 游戏内部实现：敌人AI状态机、寻路、碰撞、攻击判定、投射物边界、隐藏冷却、实际随机抽取代码。
- 版本差异：如果目标是复刻某个Steam/移动端具体版本，需要对照该版本游戏文件或实测，而不是只用当前Wiki。
- 个别页面口径冲突：例如道具数量，必须保留“页面正文/主表/模板”三种证据口径。

## 数据挖掘路径

当前快照已保存 `Modding_Notes` 页面。该页说明Brotato可通过GDRETools等Godot工具从本地游戏包恢复项目，用于modding和查看游戏内部数据。完整复刻所需的刷怪脚本、运行时代码权重和边界条件，应优先从合法拥有的本地游戏文件中提取。

2026-05-11 本机检查结果：未在常见Steam目录或Spotlight索引中发现Brotato安装包。因此本次只能完成公开Wiki快照和公开资料核验，不能直接做游戏文件级数据挖掘。

社区资料中有 ArosRising 的 wave/enemy guide 声称来自游戏文件，能作为后续交叉验证线索；但它不是当前标准口径，且不能替代本地游戏文件或Wiki原始模板。
