# Brotato数值设计

## 数值设计理念

Brotato的数值设计是其成功的关键因素之一，体现了精妙的平衡性和策略深度。

## 核心数值系统

### 角色属性系统

#### 1. 生命相关属性

##### 最大生命值（Max HP）
- **作用**：决定角色生存能力
- **重要性**：★★★★★（最高优先级）
- **获取方式**：
  - 角色初始值
  - 升级提升
  - 道具购买
  - 特殊道具
- **策略考量**：
  - 必须优先提升的属性
  - 所有构筑的基础
  - 生命值是生存的基础保障

##### 生命恢复（HP Regeneration）
- **作用**：每5秒恢复固定生命值
- **机制**：
  - 基础：每5秒恢复1点
  - 数值越高，恢复频率越快
  - 例：恢复值为5时，每1秒恢复1点
- **重要性**：★★★★☆
- **策略考量**：
  - 提升持续作战能力
  - 配合高生命值效果更好
  - 后期面对高伤害敌人的关键

##### 生命偷取（Lifesteal）
- **作用**：攻击时有概率恢复生命
- **机制**：
  - 数值为1 = 1%概率恢复1点生命
  - 数值为100 = 100%概率恢复1点生命
  - 受攻击次数影响
- **重要性**：★★★★☆
- **策略考量**：
  - 建议点满（100点）
  - 配合高攻速武器效果显著
  - 可以实现"吸血流"构筑

##### 护甲（Armor）
- **作用**：减少受到的伤害
- **机制**：百分比减伤或固定值减伤
- **重要性**：★★★★☆
- **策略考量**：
  - 后期对抗高伤害敌人必备
  - 配合生命恢复提升生存能力
  - 递减效应明显

##### 闪避（Dodge）
- **作用**：概率完全闪避攻击
- **重要性**：★★★☆☆
- **策略考量**：
  - 依赖概率，不稳定
  - 高闪避可以大幅提升生存
  - 但不如护甲稳定

#### 2. 伤害相关属性

##### 基础伤害加成（% Damage）
- **作用**：增加所有武器的伤害
- **机制**：百分比加成
- **重要性**：★★★★★
- **策略考量**：
  - 影响所有武器
  - 普适性最强的伤害提升
  - 任何构筑都需要

##### 近战伤害（Melee Damage）
- **作用**：仅增加近战武器伤害
- **机制**：百分比加成
- **重要性**：★★★★☆（近战构筑）
- **策略考量**：
  - 近战流必备
  - 混合流需要权衡
  - 纯远程流无用

##### 远程伤害（Ranged Damage）
- **作用**：仅增加远程武器伤害
- **机制**：百分比加成
- **重要性**：★★★★☆（远程构筑）
- **策略考量**：
  - 远程流必备
  - 混合流需要权衡
  - 纯近战流无用

##### 元素伤害（Elemental Damage）
- **作用**：增加元素类武器和效果伤害
- **重要性**：★★★☆☆（特定构筑）
- **策略考量**：
  - 燃烧流、法师角色核心
  - 配合元素武器和道具

##### 攻击速度（Attack Speed）
- **作用**：增加所有武器的攻击频率
- **机制**：百分比加成
- **重要性**：★★★★★
- **策略考量**：
  - 直接提升DPS
  - 配合生命偷取提升生存
  - 配合多武器效果显著
  - 可能有攻速上限

##### 暴击率（Critical Chance）
- **作用**：攻击触发暴击的概率
- **机制**：百分比概率
- **重要性**：★★★★☆
- **策略考量**：
  - 必须配合暴击伤害
  - 暴击流核心属性
  - 上限通常为100%

##### 暴击伤害（Critical Damage）
- **作用**：暴击时的伤害倍率
- **机制**：通常为1.5-2.5倍
- **重要性**：★★★★☆
- **策略考量**：
  - 需要足够的暴击率支撑
  - 爆发伤害流核心
  - 单独堆叠意义不大

#### 3. 特殊属性

##### 幸运（Luck）
- **作用**：
  - 提高商店刷出高品质物品概率
  - 增加某些特效触发率
  - 影响掉落品质
- **重要性**：★★★☆☆
- **策略考量**：
  - 前中期投资，长期收益
  - 影响整体构筑质量
  - 配合收获加速发展

##### 收获（Harvesting）
- **作用**：增加资源掉落量
- **重要性**：★★★★☆（前期）→ ★★☆☆☆（后期）
- **策略考量**：
  - 前期（1-10波）优先投资
  - 快速积累资源
  - 10波后逐步放弃，转投战斗属性

##### 工程（Engineering）
- **作用**：增强召唤物和特殊效果
- **重要性**：★★★★★（工程构筑）/ ★☆☆☆☆（其他构筑）
- **策略考量**：
  - 召唤流和工程师角色核心
  - 影响多种道具和武器效果
  - 非相关构筑基本无用

##### 移动速度（Speed）
- **作用**：提高角色移动速度
- **单位**：百分比加成（影响基础像素/秒）
- **基础速度**：约100-200像素/秒
- **重要性**：★★★☆☆
- **策略考量**：
  - 提升走位能力
  - 近战流可能更需要（需要接近敌人）
  - 过高可能难以控制（约+150%以上）
  - 参考：敌人速度50-250像素/秒，需要比快速敌人更快才能风筝

##### 射程（Range）
- **作用**：增加武器攻击距离
- **单位**：像素
- **机制**：
  - 对远程武器：+X = 射程+X像素
  - 对近战武器：+X = 射程+X/2像素（减半），且**降低攻速**
- **重要性**：★★★☆☆（远程流）/ ★☆☆☆☆（近战流）
- **策略考量**：
  - 远程流堆Range可全屏攻击
  - 近战流不应堆Range（会降低DPS）
  - 最小范围25像素
  - 参考数值：屏幕宽约960像素，半屏约480像素

## 升级属性选择系统 ⭐ 新增

> **来源**：Brotato Wiki 验证

### 基本机制

- **每次升级提供 4 个随机属性选项**
- **从 16 种属性中随机抽取**（不重复）
- **选择 1 个进行强化**
- **属性有 4 个品质等级（Tier I-IV）**，数值不同

### 升级属性刷新 ⭐ Wiki确认

**升级时可以花费材料刷新 4 个属性选项**：

| 规则 | 说明 |
|------|------|
| **刷新价格** | 与商店刷新使用**相同公式** |
| **独立计价** | 升级刷新和商店刷新**分别计算次数**，互不影响 |
| **同波次共享** | 同一波次获得多次升级时，刷新价格**共享递增** |
| **跨波次重置** | 下一波次升级时，刷新次数重置 |

**刷新价格公式**（与商店相同，1.1.4.0版本后）：
```
首次刷新 = floor(波次 / 2)
第N次刷新 = 首次费用 + (N-1) × floor(波次 × 0.33)
```

> **Wiki原文**：*"Rerolling is also possible when the character levels up and is offered a selection of upgrades. This rerolling uses the same prices as the Shop."*
>
> *"The price for rerolls resets between level-ups and the shop, so rerolling your level-ups doesn't affect the cost of rerolling in the shop."*

### 幸运（Luck）对升级品质的影响

- 幸运属性使用**与商店物品相同的公式**
- 在非保底等级，幸运越高，出现高品质的概率越大
- 例：6-9级时，高幸运会提高 Tier II 出现概率

### 设计意义

1. **成长曲线平滑**：前期低品质，后期高品质，匹配难度曲线
2. **幸运属性多用途**：不仅影响商店，还影响升级品质
3. **里程碑获得感**：5/10/15/20/25 级有明确的品质提升节点
4. **随机中的确定性**：保底机制防止运气过差
5. **刷新机制**：提供"花钱换选择"的策略深度

### 属性成长曲线

#### 线性成长
- 最大生命值
- 基础伤害加成
- 大部分属性

#### 阈值效应
- **生命偷取**：建议点满100点达到100%触发
- **暴击率**：达到100%后不再需要提升
- **闪避**：可能有上限

#### 边际递减
- **护甲**：堆叠过多，每点收益递减
- **攻击速度**：可能有攻速上限
- **移动速度**：过高反而难以操控

### 属性公式与上限详解 ⭐ Wiki确认

#### HP Regeneration（生命回复）- 正确公式

**常见误解**：每5秒恢复固定生命值

**实际机制**（Wiki确认）：
```
第1点 HP Regen = 0.20 HP/秒（持续恢复）
后续每点 = +0.089 HP/秒（递减效应）

计算示例：
- 1点 HP Regen = 0.20 HP/秒
- 5点 HP Regen = 0.20 + 4×0.089 = 0.556 HP/秒
- 10点 HP Regen = 0.20 + 9×0.089 = 1.001 HP/秒
- 20点 HP Regen = 0.20 + 19×0.089 = 1.891 HP/秒
```

> ⚠️ **关键**：是持续恢复（每秒），不是间歇恢复（每5秒）

#### Life Steal（生命偷取）- 触发上限

```
机制：角色生命偷取 + 武器基础生命偷取 = 总触发概率
上限：每0.1秒最多触发1次 = 最大 10 HP/秒

示例：
- 100%生命偷取 + 攻速10/秒 = 每秒10次触发 = 10 HP/秒
- 超过上限的攻击不会额外回血
```

#### Attack Speed（攻击速度）- 硬上限

```
硬上限：最快每秒12次攻击
近战武器：有独立的更低上限
递减效应：高数值时每点加成效果降低
负值计算：与正值使用不同公式
```

#### Dodge（闪避）- 角色差异上限

| 角色 | 闪避上限 |
|------|----------|
| 普通角色 | 60% |
| Cryptid | 70% |
| Ghost | 90% |

> 负闪避视为0%处理

#### Range（射程）- 最小值与冷却

```
最小射程：25像素
近战特殊：射程增加会增加冷却时间（更长的攻击动画）
近战减半：+X射程 = 实际+X/2像素
```

### 属性负值效果 ⭐ Wiki确认

当属性降为负值时的特殊处理：

| 属性 | 负值效果 |
|------|----------|
| **Max HP** | 视为1处理（最低1血） |
| **HP Regeneration** | 视为0处理（不会扣血） |
| **Life Steal** | 降低触发概率，最低0% |
| **Damage** | 每点-1%伤害，武器最低伤害为1 |
| **Melee Damage** | 降低近战基础伤害，最低1 |
| **Ranged Damage** | 降低远程基础伤害，最低1 |
| **Elemental Damage** | 降低元素基础伤害，最低1 |
| **Attack Speed** | 攻击变慢，使用不同计算公式 |
| **Crit Chance** | 从武器基础暴击中减去，最低0% |
| **Engineering** | 降低结构伤害/治疗量，最低1 |
| **Range** | 减少射程（近战减半），最小25像素 |
| **Armor** | 变为**增加**受到的伤害x% |
| **Dodge** | 视为0%处理 |
| **Speed** | 移动变慢，-100%时完全静止 |
| **Luck** | 降低掉落概率和商店品质 |
| **Harvesting** | 失去材料/经验，5%增量机制不生效 |

> ⚠️ **护甲负值特别重要**：负护甲会让你**受到更多伤害**，而非减少！

### Curse属性 ⭐ DLC内容（Abyssal Terrors）

**来源**：DLC "Abyssal Terrors"

#### 基础机制

Curse属性会给敌人添加诅咒，被诅咒的敌人更强但掉落更多：

| 诅咒敌人效果 | 数值 |
|--------------|------|
| 伤害增加 | +25% |
| 速度增加 | +15% |
| 生命值增加 | +150% + 每点Curse额外+2% |
| 材料掉落增加 | +33% |

#### 概率上限

| 效果 | 上限 |
|------|------|
| 诅咒敌人生成概率 | 最高50% |
| 诅咒道具生成概率 | 最高15% |
| 生命缩放上限 | 300 Curse时达到最大 |

#### 商店效果

- 商店有概率出现带**诅咒增强**的道具
- 诅咒道具属性更强，但会增加Curse

#### 特殊规则

- **不能为负值**：Curse最低为0
- **风险收益**：高Curse = 高风险高回报

## 数值平衡机制

### 1. 正负属性平衡

许多强力效果都伴随负面属性：
- **蛇**：燃烧扩散效果 / -1最大生命
- **小象**：拾取材料时造成伤害 / -1基础伤害
- **角斗士角色**：+120%攻速 / -40%基础攻速 + 仅近战

**设计目的**：
- 避免必选道具/角色
- 增加选择深度
- 鼓励多样化构筑

### 2. 资源约束

#### 材料限制（Materials）
- 材料总量有限
- 无法购买所有道具
- 需要优先级判断

#### 属性点限制
- 升级获得的属性点有限
- 需要在多种属性间取舍

#### 武器槽位限制
- 最多6把武器
- 需要选择最优组合

### 3. 时间压力

#### 波次递进
- 20波结构
- 敌人强度持续提升
- 需要在有限波次内完成构筑

#### 商店刷新成本
- 每次刷新材料消耗递增
- 无限刷新但成本越来越高
- 防止无脑刷新最优商品

## 敌人数值设计

### 怪物数值结构

#### 基础数值
- **作用**：保证前期怪物的表现
- **设计**：前5波的怪物基础属性
- **目的**：新玩家也能正常游戏

#### 成长数值
- **作用**：保证20波中怪物随玩家成长
- **设计**：每波怪物属性递增
- **目的**：保持难度平行上升

### 数值与数量平衡

**核心原则**：
- **低数值怪物**：生成数量多
- **高数值怪物**：生成数量少

**实例**：
- 普通小怪：低血量、低伤害、大量
- 精英怪：中等血量、中等伤害、中量
- Boss级：高血量、高伤害、少量

**目的**：
- 创造不同的威胁模式
- 形成目标优先级
- 增加策略深度

### 难度曲线

```
波次    怪物强度    玩家强度    相对难度
1-5      低         低         容易
6-10     中         中         适中
11-15    高         高         挑战
16-19    很高       很高       困难
20       极高       极高       Boss
```

**设计目标**：
- 保持平行上升
- 避免难度断层
- 后期适度增加压力

## 经济系统数值

### 资源获取

#### 材料（Materials）掉落（唯一货币）
- **基础掉落**：每个怪物固定掉落（不同敌人差异很大）
- **收获加成**：收获属性提升材料掉落量
- **波次加成**：后期波次更高密度/更高收益
- **用途**：购买武器、道具、属性提升，以及刷新商店（同一货币）

#### 经验值掉落
- 击败敌人获得
- 累积升级

### 商店经济

#### 物品定价
- **普通品质**：便宜
- **稀有品质**：中等
- **史诗品质**：较贵
- **传说品质**：昂贵

**平衡原则**：
- 价格与强度对等
- 但性价比各不相同
- 鼓励策略选择而非无脑购买最贵

#### 刷新成本
- **首次刷新**：低成本
- **后续刷新**：递增成本
- **目的**：
  - 允许有限刷新
  - 防止无限刷新
  - 增加决策压力

### 资源规划

#### 前期（1-5波）
- **重点**：
  - 投资收获属性
  - 建立基础生存能力
  - 积累资源
- **经济策略**：滚雪球

#### 中期（6-10波）
- **重点**：
  - 明确构筑方向
  - 平衡收获与战斗属性
  - 购买核心道具
- **经济策略**：转型期

#### 后期（11-15波）
- **重点**：
  - 减少收获投资
  - 全力提升战斗属性
  - 完善构筑
- **经济策略**：成型期

#### 终局（16-20波）
- **重点**：
  - 极限强化核心机制
  - 补足短板
  - 应对最强敌人
- **经济策略**：冲刺期

## 数值设计启示

### 1. 多维度属性系统
- 不是单一的"战力"
- 多种属性提供策略深度
- 鼓励差异化构筑

### 2. 取舍机制
- 资源有限
- 无法全面发展
- 需要明确方向和优先级

### 3. 阶段性目标
- 不同阶段有不同的属性重点
- 收获：前期→中期→后期（递减）
- 战斗属性：前期→中期→后期（递增）

### 4. 平衡的艺术
- 没有绝对最强的属性
- 配合和协同是关键
- 边际效应防止无脑堆叠

### 5. 代价与回报
- 强力效果伴随代价
- 增加选择深度
- 避免必选项
